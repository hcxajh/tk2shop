## 步骤 3：点击左侧 `Online Store`

### 正确点法
不是点外层容器，也不是点页面正文里同名文本。

当前已确认的正确定位：
- 在 `ul._Section_1r2u9_1` 里
- 点击**第一个 `a` 标签**

### 错误示范（已踩坑）
1. 按页面全文匹配 `Online Store`
   - 会误点到页面正文里的同名文本
   - 师父侧看不到左侧导航变化

2. 点左侧 `Online Store` 外层 `li`
   - 可能命中容器，不触发真正切换

### 正确结果
- 点击后会进入主题相关页面
- 已实际跳转到：
  - `https://admin.shopify.com/store/vellin1122/themes?...`

### 脚本化注意点
- 左侧栏操作不能只靠纯文字匹配
- 必须优先定位到侧栏真实链接节点
- 当前这一步最稳的规则是：
  - `ul._Section_1r2u9_1 > 第一个 a`

## 步骤 4：识别 Shopify 主题页的右侧主体区域

### 关键结论
Shopify 主题页的右侧主体内容，不在外层主文档里。

### 当前确认
- 顶层文档主要是左侧导航外壳
- 右侧主体在一个大 iframe 里

### 已识别到的 iframe 特征
- 类名：`_Frame_1vogr_1`
- 大尺寸 iframe
- 内容源来自：`online-store-web.shopifyapps.com/themes...`

### 错误示范（已踩坑）
- 只在外层页面 DOM 里找 `Edit theme`
- 结果只能拿到左侧导航，看不到右侧主题卡片主体

### 脚本化注意点
- 进入主题页后，后续元素定位前要先切进右侧大 iframe
- 不切 iframe，后面所有右侧主体元素定位都会偏

## 步骤 5：进入 Theme Editor（主题编辑器）

### 当前探索结论
在主题页主体 iframe 中，存在 `Edit theme` 入口。

### 已确认事实
1. 右侧主体中确实能读到：
   - `Current theme`
   - `Edit theme`
   - `Publish`

2. `Edit theme` 周边真实结构里，已经抓到明确编辑器链接：
   - `https://admin.shopify.com/store/a13xn8-v5/themes/155252195550/editor`

3. 直接打开这个链接，已经成功进入主题编辑器页。

### 当前最稳做法
在自动点击还没完全稳之前，现阶段最稳的进入方式是：
- 直接打开已抓到的编辑器链接

### 错误示范（已踩坑）
1. 在外层 DOM 里直接找 `Edit theme`
   - 找不到右侧主体

2. 在 iframe 里泛化“往上找可点击父节点”
   - 容易误命中整页大容器 `div`
   - 不会真正触发跳转

3. 只用 `closest('a')` 去找
   - 有时命中不到真实结构
   - 但从实际抓到的按钮组 HTML 看，`Edit theme` 旁边确实存在真实 `a href=".../editor"`

### 当前阶段推荐记录方式
进入主题编辑器的可靠记录先写成：
1. 进入右侧主题 iframe
2. 读取 `Edit theme` 按钮组区域
3. 拿到真实编辑器链接
4. 直接打开编辑器链接

### 后续待补强
- 继续把“从 iframe 内直接精确点击 `Edit theme` 链接”做稳
- 最终替代“直接打开抓到的 href”方案

## 步骤 6：Theme Editor 内切到 Product 并创建模板

### 当前已确认路径
1. 先进入 Theme Editor 主体 iframe
2. 点击顶部模板切换按钮：
   - `[aria-controls="topbar-picker-templates"]`
3. 在模板列表里点击：
   - `li#PRODUCT button`
4. 再点击：
   - `li#__create__ button`
5. 在弹窗里输入模板名
6. 点击右下角 `Create template`

### 关键结论
- 不能点 `Product` 文字本身
- 不能只点 `li#PRODUCT` 容器
- 必须点里面真正的 `button`

### 输入框注意点
- 模板名输入框是受控输入，不是简单 `input.value = xxx` 就稳
- 直接脚本赋值后，值可能回弹为空
- 应优先使用真实输入事件
- 如果输入结果和期望不一致，应该报错停住，不要假装继续成功

## 步骤 7：进入 Multicolumn

### 当前已确认路径
- 先锁定：
  - `li[data-component-extra-section_type="multicolumn"]`
- 再点击这一行里的主按钮：
  - `button.Online-Store-UI-NavItem__PrimaryAction_1232d`

### 成功判定
进入后至少会看到以下其一：
1. 左侧出现：
   - `Add Column`
   - `Column`
   - `Column`
   - `Column`
2. 设置区出现：
   - `Heading`
   - `Image`
   - `Button`
   - `Layout`

### 错误示范（已踩坑）
- 点 `Multicolumn` 文字 `span`
- 点 `Hide section`
- 点到旁边模块的主按钮

### 结论
- 主题编辑器左侧模块树，不能靠文字节点乱点
- 最稳规则是：
  1. 先找对应 `li`
  2. 再点该行主按钮

## 当前已经确认的关键坑位

### 1. Shopify 后台很多右侧主体不在顶层 DOM
- 必须先判断是否在 iframe 里
- 否则只会拿到左侧导航外壳

### 2. 不能只按文本乱点
- 同名文本在正文、侧栏、容器里都可能出现
- 只按文本匹配非常容易误点

### 3. 不能把“可点击父节点”范围放太宽
- 容易直接爬到整页大容器
- 看似点击成功，实际页面没变化

### 4. 导入成功后不用再单独重开后台
- 当前真实链路已经修正为：
  - 导入成功 → 确认产品进店 → 直接从当前后台上下文进入主题编辑
- 后续脚本设计必须遵守这一点
